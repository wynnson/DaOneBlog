Source: GeeksforGeeks

Primer: Largest Rectangle in Histogram

The problem reads as follows: Given an array of integers heights representing the histogram's bar height where the width of each bar is 1, return the area of the largest rectangle in the histogram.

This is also Leetcode 84. I will refer to the individual rectangles as bars.

Before I show you a solution, try to solve it yourself!

Brute-Force

We will start with the most obvious approach. The intuition is that we enumerate over all bars with 2 pointers. We will need to keep a lowerbound height as we loop through since we are constrained by it.


Here is an example on iteration i = 0:

The idea is that for each bar, we expand our bounds to see the maximum area rectangle we can make. In code this looks like: 

def largestRectangleArea(self, heights: List[int]) -> int:
    n = len(heights)
    max_area = 0

    for i in range(n):
        lower_bound = math.inf
        for j in range(i, n):
            lower_bound = min(lower_bound, heights[j])
            rectangle = lower_bound * (j - i + 1)
            max_area = max(max_area, rectangle)

    return max_area

Note on +1

We need the +1 for inclusivity. For example, in the case that i = 0, j = 0, doing j - i = 0 - 0 = 0. The +1 is there to say that the bar actually has a width of 1. Another way to think about it is the +1 shifts the j pointer outside the bar (exclusive) so that when you subtract j - i, the difference is the width of the rectangle.

The time complexity will be $O(n^2)$ from the enumeration of the bars (i, j). This will be a bottleneck on large input sizes. Since we didn't allocate extra space, the space complexity is constant, $O(1)$.

Div. & Conq.

A powerful idea is that the problem is comprised of subproblems. This is known as Divide and Conquer. We can break our max area down into 3 subproblems:

  1. The max area is the maximal width times the shortest bar
  2. The max area is confined by the left bar
  3. The max area is confined by the right bar


Here is a visual provided by the LeetCode editorial:

Source: Leetcode 84 Editorial

The core idea [divide step] is that we can keep splitting based on the minimum. Then when we recurse back [conquer step], we can return the maximum of the 3 cases: $$ \max( \text{left split area}, \ \ \text{right split area}, \ \ \text{min height $\times$ width} ) $$ This way, our final returned solution contains the max area of the entire histogram. 

            
                def recurse(self, heights: List[int], i: int, j: int) -> int:
                    # base case
                    if i > j:
                        return 0

                    # get min idx (argmin)
                    min_idx = i
                    for k in range(i, j + 1):
                        if heights[k] < heights[min_idx]:
                            min_idx = k

                    # divide (split)
                    left = self.recurse(heights, i, min_idx - 1)
                    right = self.recurse(heights, min_idx + 1, j)

                    # conquer
                    return max(
                        heights[min_idx] * (j - i + 1),
                        left,
                        right
                    )

                def largestRectangleArea(self, heights: List[int]) -> int:
                    return self.recurse(heights, 0, len(heights) - 1)

The time complexity on average is $O(nlogn)$ from the splitting, where $n$ is the length of the heights array. Worse case is $O(n^2)$ if the heights array was already sorted (meaning finding the min doesn't matter). Space complexity is $O(n)$ because of the recursion.

Masters Thereom (Div & Conq)

The masters theorem gives the runtime of divide and conquer algorithms. This is given by: $$ T(n) = a \cdot T(\frac{n}{b}) + f(n) $$ where $a \ge 1, b > 1$, $f(n) = d$.

  • $a$ represents # of subproblems
  • $b$ represents how much the subproblem shrinks (division factor)
  • $f(n)$ represents any extra "work." This is represented by $d$.

In this case, we split into left and right so $a = 2$. $b = 2$ because left and right divides our subproblem space in half. $f(n) = n$ because finding the argmin (min idx) is a linear scan. $$ T(n) = 2 \cdot T \left( \frac{n}{2} \right) + O(n) $$ The runtime is given by the following cases: $$ T(n) = aT\left(\frac{n}{b}\right) + d $$ $$ T(n) \in \begin{cases} \Theta(n^d), & a < b^d \\ \Theta(n^d \log n), & a = b^d \\ \Theta(n^{\log_b a}), & a > b^d \end{cases} $$ We have the case: $a = b^d$ since $2 = 2^1$. $d = 1$ meaning linear work ($d = 0$ is constant) done. Thus, the runtime is: $$ O(n^1 log n) = O(nlogn) $$

Monotonic Stack

A monotonic stack is simply a stack with either increasing or decreasing ordering. In other words, the stack is ordered / sorted (can be strictly increasing / decreasing as well). To maintain ordering, we use a monotonically increasing stack that is not strictly increasing. This ordering is known as an invariant meaning rule or constant. The monotonic stack method is a standard way to solve problems relating to:

  • next greater
  • previous greater
  • next smallest
  • previous smallest
  • span / range

Here is an awesome video by AlgoMonster on monotonic stacks.

This is no longer monotonic so we must continuously pop to fix the ordering:
$s$ = [1, 3, 2], popping 5

Rectangles and Monotonic Stack

Here is the core idea: the largest rectangle exists in the increasing sections. That means we can use an increasing monotonic stack. To make this make a little more sense, look at the second 2. Upon discovering 2, we introduced another lowerbound (where the first element (1) of the monotonic stack was the previous lowest bound). Thus, we should test the candidate rectangle (in red) for a max and remove parts of it to maintain our invariant.

At the second 2, $s$ = [1, 5, 6, 2] meaning we have to pop since we introduced a concavity. After popping we remain with $s$ = [1, 2] and the algorithm continues, leaving us with the blue section. But notice, at the end, we still have $s$ = [1, 2, 3]. We never check the blue rectangle since the invariant is maintained. To get around this we can do 2 things: post-processing or a nifty trick :) We can just pad the end with a 0 to force concavity, clearing our stack!

Post Processing Code (If Interested)

        
            while stack:
                current_height = heights[stack.pop()]
                current_width = len(heights) - stack[-1] - 1
                max_area = max(max_area, current_height * current_width)

Note

We will use the 0 trick to force the invariant.

Overview

To tie it all together, here is the entire idea:

  1. Loop through the bars
  2. While current height is less than the top of the stack, pop and calculate max height
  3. Otherwise, append to stack (we will just append the idx)
  4. Return max height

Popping Mechanic

Before I implement it, we should understand the popping mechanic in a little more depth. When I say we pop from the stack, that means we are popping from the largest bar (stack top). In other words we are calculating the rectangle from right to left. To properly find the width, we will use the last valid left neighbor$^*$ (maintaining the invariant) for a left bound and the current index position (where we start popping bc of invariant) as the right bound. This means we will have to subtract 1 as both ranges are exclusive and subtracting 1 will make right bound inclusive (we can also add 1 to left bound). This way, we can calculate the width properly. So our update for max area looks like this (note we could have used idx - 1 - left): $$\max(heights[top] * (idx - (left + 1)), max \ area)$$

Edge Case

$^*$ Suppose we have $heights$ = [0, 3, 2, 5, 0], $s$ = [0, 2, 5], and our current iteration points to the second 0, meaning we have to clear our stack. The important thing to note is that when popping 2, we use the left bound as 0 (the last valid left neighbor). We can't just use 2's neighbor (3) or else we would be missing a bar when we do +1 or -1.

This shows the popping mechanic when we reach the second 2. Note how we check the rectangles R to L. 

        
            def largestRectangleArea(self, heights: List[int]) -> int:
                if len(heights) == 0:
                    return 0

                heights.append(0)
                stack = []
                max_height = 0
                
                for idx, num in enumerate(heights):
                    while stack and num < heights[stack[-1]]:
                        top = stack.pop()
                        left = stack[-1] if stack else -1
                        max_height = max(
                            heights[top] * (idx - (left + 1)),
                            max_height
                        )
                    stack.append(idx)
                
                return max_height
In this case I used left = stack[-1] if stack else -1. This is equivalent of putting a placeholder bar before everything so our width calculation works. There are also other ways to do this.

This runs in $O(n)$ since we only do one pass. It uses $O(n)$ space for the stack. Thus this is the most effective solution.
